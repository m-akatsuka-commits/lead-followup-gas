# ナレフル追客管理システム Readme
## 1. システム概要
本システムは、スプレッドシート上の顧客データに基づき、適切なタイミングで追客メール（2通目〜4通目）を送信するための管理ツールです。
Google Apps Script (GAS) を利用したWebアプリケーションとして動作し、送信ミス防止のための「プレビュー機能」や、効率化のための「一括送信機能」を備えています。

### 構成
- **管理シート**: `2026.01以降`（顧客データ・送信ステータス管理）
- **設定シート**: `設定`（メール文面・署名・担当者マッピングの管理）
- **実行環境**: Google Apps Script (Webアプリケーション)

---

## 2. メンバー増員時のメンテナンス手順（重要）
新しく担当者が増えた場合、以下の**3箇所**を更新する必要があります。

### ① スプレッドシート「設定」シートの更新
「設定」シートの **B15以降** に、新しい担当者の情報を追加してください。
- **偶数行（B15, B17...）**: `氏名_署名`（メール署名内に表示されるフルネーム）
- **奇数行（B16, B18...）**: `氏名_Mail`（返信先アドレスとして設定されるメールアドレス）

※ 既存の最終行（現在はB36まで）の続きに、項目名（A列）と値（B列）をセットで追記してください。

### ② `Code.gs` の修正
GASエディタを開き、`getStaffInfoFromSheet` 関数内のマッピングを更新します。

1. **取得範囲の変更**:
   `const values = settingSheet.getRange("B15:B36").getValues();` の `B36` を、追加した最終セルに合わせて拡張してください。
2. **キーリストの追加**:
   `const staffKeys = ["西山", "伊藤", ..., "時田"];` の配列の末尾に、スプレッドシートのJ列に入力される**「苗字のみ」**を追加してください。
   *※スプレッドシートの並び順と一致させる必要があります。*

### ③ `index.html` の修正
管理画面（UI）のプルダウンメニューに新しい名前を追加します。

1. `index.html` 内の `<select id="globalStaffSelect">` タグを探します。
2. `<option value="時田">時田 温子</option>` の下に、新しい担当者の行を追加してください。
   ```html
   <option value="苗字">フルネーム</option>
   ```

## 3. スプレッドシートの構成

### メインシート（2026.01以降）
| 列 | 項目名 | 役割 |
|:---|:---|:---|
| B列 | 商談獲得 | 「商談獲得」と入力されると自動送信・リストから除外 |
| D列 | 顧客区分 | リンク（最上位/上位/一般）の判定に使用 |
| J列 | 初回対応者 | **システムのキー。** ここに記載された「苗字」を基に署名等を生成 |
| K列 | 種類 | 「リスキリング研修」判定に使用 |
| L〜O列 | 各メール送信日 | 送信完了時に `MM/dd` 形式で自動記入 |
| AB列 | 追客終了フラグ | チェック（TRUE）が入るとリストから除外 |

### 設定シート
| セル | 役割 |
|:---|:---|
| B2〜B7 | 通常追客（2〜4通目）の件名・本文 |
| B8 | 共通署名テンプレート（`<J列>`などの変数が置換されます） |
| B9〜B14 | リスキリング研修専用 追客（2〜4通目）の件名・本文 |
| B15〜 | 各担当者のフルネームおよびメールアドレスマッピング |

## 4. 運用ルールと仕様
- **送信元（From）**: 
  GASの仕様上、**「Webアプリを実行したユーザー自身」**のアドレスから送信されます。
- **Reply-To（返信先）**: 
  UI上のプルダウンで選択した担当者のアドレスが設定されます。
- **名乗り（本文冒頭）**: 
  プルダウンで選択した担当者の「苗字のみ」が反映されます。
- **非表示ロジック**: 
  「商談獲得済」または「AB列にチェックがある行」は、リスト読込時に自動的に除外されます。
- **追客ステップ**: 
  1通目（手動）に基づき、本アプリでは「追客2（旧追客1）」「追客3」「追客4」として処理します。

## 5. デプロイ時の注意点
メンバー全員が自分のアドレスで送信できるようにするため、更新時は以下の設定でデプロイしてください。

1. **次のユーザーとして実行**: `ウェブ アプリケーションにアクセスしているユーザー`
2. **アクセスできるユーザー**: `CLINKS株式会社 内の全員`

## 6. トラブルシューティング
- **「Script function not found: doGet」と出る**:
  デプロイが正しく完了していないか、URLが古い可能性があります。「新しいデプロイ」からURLを再生成してください。
- **名前が「担当者」になってしまう**:
  J列の苗字と `Code.gs` 内の `staffKeys` の文字列が完全一致しているか確認してください（スペースの有無など）。
- **特定の行が表示されない**:
  商談獲得列に文字が入っていないか、AB列にチェックが入っていないか確認してください。
